## 第1题：Claude Code 到底是什么？它和网页版 Claude 有什么本质区别？

### 核心结论

**Claude Code 不是"带了文件访问权限的Claude"，它是一个"会使用工具的 AI Agent（智能体）"。**

它通过"工具调用"机制，把一个"只会说话的AI"变成了"能动手的AI程序员"。

### 三个本质区别

| 维度 | 网页版 Claude | Claude Code |
|------|--------------|-------------|
| **运行位置** | 浏览器里，图形界面 | 本地电脑终端里，命令行界面 |
| **能否看到文件** | 看不到。只能读你主动贴进对话框的内容 | 能看到。能直接读取项目文件夹里的所有文件 |
| **能否动手操作** | 只能说。告诉你答案，你自己去做 | 能动手。能直接修改文件、执行命令、运行代码、提交到 Git |

### 工具调用机制（Claude Code 的核心）

Claude 模型本身不能直接读文件或执行命令，它需要通过"工具"来做。Claude Code 给 Claude 模型装备了一套"工具箱"：

| 工具名称 | 作用 | 对应的底层操作 |
|---------|------|---------------|
| Read | 读取文件内容 | 类似 `cat 文件名` |
| Glob / LS | 列出目录和文件 | 类似 `ls` |
| Bash | 执行任意终端命令 | 直接运行 bash |
| Grep | 在文件里搜索内容 | 类似 `grep` |
| Edit / Write | 修改或创建文件 | 直接写文件 |
| WebFetch | 抓取网页内容 | 类似浏览器 |

### 工具调用的完整流程

当你问 Claude Code 一个问题时，它的内部流程是：

```
1. 分析你的需求
2. 决定需要用哪些工具
3. 调用工具去获取真实信息
4. 把工具返回的结果读进自己的上下文
5. 基于真实信息生成回答
```

### 关键观察：工具调用日志

在 Claude Code 界面里，按 `Ctrl + O` 可以展开工具调用的详细日志，看到类似：

```
● Read(/Users/yinlei/yinlei-project/hello.py)
  L Read 2 lines

● Read(/Users/yinlei/yinlei-project/greeting.py)
  L Read 8 lines
```

每次工具调用前的**绿色圆点**代表成功执行，红色代表失败。

### 适用场景对比

**Claude Code 最适合的场景：**

- 在真实项目里写代码（它能读懂现有代码结构）
- 修复 bug（贴报错，它直接去改）
- 重构代码（多文件协同修改）
- 代码审查和解释（自动读整个项目）
- 执行一系列命令（一次性做完多个操作）
- 学习陌生代码库

**Claude Code 不适合的场景：**

- 查基础语法（用网页版或搜索更快）
- 闲聊和创意写作
- 不涉及代码的通用问题

### 一句话总结

> **网页版 Claude 是一个聪明的"顾问"，给你建议和答案。**
> **Claude Code 是一个聪明的"实习生"，直接帮你干活。**

**Agent（智能体）的定义：** 一个能感知环境、做出决策、并执行动作的 AI 系统。Claude Code 就是一个 Agent。

## 第2题：CLAUDE.md 文件是干什么的？为什么它是 Claude Code 的灵魂？

### 核心结论

**CLAUDE.md 是你给 Claude Code 写的"项目说明书"。** 它是一个 Markdown 格式的文件，放在项目根目录，Claude Code 启动时会自动读取它，把里面的内容当作"基础知识"记住。

### 为什么需要 CLAUDE.md

**Claude Code 有两个身份：**

- 一个聪明的实习生：它会写代码
- 一个失忆症患者：每次对话开始，它都不记得你的项目是什么样

**问题：** 没有 CLAUDE.md 的话，每次你想让它写代码，它都会有很多"该怎么做"的疑问——用什么语言？文件放哪？命名规则？代码风格？不告诉它就只能猜，而且每次猜的可能不一样。

**解决方案：** 给它一份"项目规范文档"放在它桌上，让它每次上岗前先看一遍。这就是 CLAUDE.md 的设计思路。

### CLAUDE.md 的两种级别

#### 1. 用户级 CLAUDE.md（全局的）

- **位置：** `~/.claude/CLAUDE.md`
- **作用域：** 对你所有项目都生效
- **用途：** 写你的个人偏好，不属于某个具体项目
- **例子：** "我希望 Claude 用中文回复"、"我是新手，解释代码时从基础讲起"

#### 2. 项目级 CLAUDE.md

- **位置：** `./CLAUDE.md`（项目根目录下）
- **作用域：** 只对这个项目生效
- **用途：** 写项目特有的规则
- **例子：** "这个项目用 Python 3.14"、"所有文件放在 `src/` 文件夹下"
- **重要特性：** 会被提交到 Git，让所有协作者共享同一份项目规范

### 判断规则写在哪里的口诀

> "这条规则是跟着**我**走，还是跟着**项目**走？"
>
> - 跟着我走 → 用户级
> - 跟着项目走 → 项目级

### 加载机制

**Claude Code 启动时会自动读取 CLAUDE.md，加载顺序：**

```
1. ~/.claude/CLAUDE.md              ← 用户级（全局偏好）
2. yinlei-project/CLAUDE.md         ← 项目根目录
3. yinlei-project/frontend/CLAUDE.md ← 当前工作目录（如果在子目录）
```

**三份会按顺序叠加。** 越靠下的优先级越高——如果有冲突，以更"近"的那份为准。

### 重要细节：读取时机

**CLAUDE.md 只在 Claude Code 启动那一次被读取。** 如果你在对话过程中手动修改了 CLAUDE.md 文件，Claude Code 不会自动察觉到修改。

**让 Claude Code 刷新记忆的方法：**

- **方法 1（暴力但可靠）：** `/exit` 退出，然后重新启动 `claude`
- **方法 2（优雅）：** 使用 `/memory` 命令，进入面板编辑后自动刷新

### CLAUDE.md 里应该写什么

**应该写的（4类）：**

1. **项目背景**：项目是干什么的
2. **技术栈**：用了什么语言、工具、版本
3. **代码规范**：你希望 Claude Code 怎么写代码
4. **项目结构**：文件放在哪、命名规则

**还可以写"给 AI 的行为指令"：**

```markdown
## 给 Claude Code 的指令
- 当我犯明显错误时，直接指出，不要客气
- 我是 Python 新手,解释代码时不要跳过基础概念
- 优先给出最简单的实现,不要引入复杂的设计模式
- 每次修改文件前,先告诉我你要改什么
```

**不应该写的（避免以下坑）：**

- 冗长的项目历史
- 个人情绪和抱怨
- 教程链接
- 过于具体的代码示例

### 常见的坑和最佳实践

#### 坑 1：写太多

**问题：** CLAUDE.md 内容会占用 Claude 的上下文空间,写太多会挤占对话空间,而且内容越多 Claude 越难抓住重点。

**建议：** 保持在 500 字以内(不包括代码示例)。只写最核心的规则,能删就删。

#### 坑 2：写得太模糊

**错误示范：**

```markdown
- 代码要写得好
- 注意代码质量
```

**正确示范：**

```markdown
- 函数必须有文档字符串
- 每个文件不超过 200 行
- 变量命名用 snake_case
- 所有异常都要显式处理,不要用裸 except
```

#### 坑 3：没有优先级

**建议：** 用显眼的标记区分必须遵守和建议遵守的规则:

```markdown
## 核心规则（必须遵守）
- ⚠️ 永远不要修改 `.git/` 文件夹里的任何内容

## 一般规范（建议遵守）
- 变量命名用 snake_case
```

#### 坑 4：和实际不符

写 CLAUDE.md 之前,先确认事实。项目实际用什么版本、什么库,就写什么。

#### 坑 5：把它当成"一次性文档"

CLAUDE.md 是一个**活的文档**。每次发现 Claude 搞错规则,回去更新 CLAUDE.md 加一条新规则。每次项目结构变化,也要同步更新。

### /memory 命令的界面说明

在 Claude Code 里输入 `/memory` 会打开记忆面板,显示:

- **Auto-memory: on/off** — 自动记忆开关
- **选项 1：User memory** — 用户级 CLAUDE.md(`~/.claude/CLAUDE.md`)
- **选项 2：Project memory** — 项目级 CLAUDE.md(`./CLAUDE.md`),标注 "Checked in" 表示会被提交到 Git
- **选项 3：Open auto-memory folder** — 打开自动记忆文件夹

### 一句话总结

> **CLAUDE.md 是 Claude Code 的"启动脑袋"——每次它上班前,先把你写的规范看一遍,然后再开始工作。学会写 CLAUDE.md,你的 Claude Code 使用体验会提升 10 倍。**
